# 第三方插件注册区
# Scripts/Packages.sh 会解析这里的 @vendor 规则，把第三方插件源码拉进编译树。
# Aurora 主题已经改成按参考项目单独处理：
# - 主题源码和配置页源码由 Scripts/Packages.sh 固定拉取
# - 默认主题切换由 Scripts/Settings.sh 处理
# - 因此这里不再为 Aurora 写 @vendor 规则
# 规则格式：
#   # @vendor <config-package>|<repo-url>|<branch>|<src-rel>:<dst-rel>[;<src-rel>:<dst-rel>...]|<hook>
# hook 可以留空、写 none，或者写 po2lmo。
# 注意：
# - luci-app-homeproxy 已经在官方 luci feed 中存在，所以这里不再写它的 @vendor。
# - luci-app-tailscale 这里改用 asvow 维护版，用来替换官方 feed 里的 community 版本。
# - 下面保留的第三方项，都是当前官方 feed 不方便直接使用或不存在的插件。
# @vendor luci-app-openclash|https://github.com/vernesong/OpenClash.git|dev|luci-app-openclash:package/luci-app-openclash|po2lmo
# @vendor luci-app-fancontrol|https://github.com/bigmalloy/luci-app-fancontrol.git|main|openwrt-feed:package/luci-app-fancontrol|
# @vendor luci-app-tailscale|https://github.com/asvow/luci-app-tailscale.git|main|.:package/luci-app-tailscale|tailscale-compat
# @vendor luci-app-mini-diskmanager|https://github.com/4IceG/luci-app-mini-diskmanager.git|main|.:package/luci-app-mini-diskmanager|
# @vendor luci-app-iperf3|https://github.com/Gevatter-Tod/luci-app-iperf3.git|main|.:package/luci-app-iperf3|
